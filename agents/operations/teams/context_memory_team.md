# Context & Memory Team

## Purpose
The Context & Memory Team is responsible for optimizing the workspace's system prompts, rule files, and memory structures to ensure optimal LLM performance and context window efficiency.

## Team Members
* **Agents Orchestrator**: Coordinates the audit and enforces rules.
* **Technical Writer**: Documents findings, refines rule sets, and synthesizes reports.

## Primary Responsibilities
1. Audit configuration files (`.clinerules`, `.roomodes`, `config/settings.md`) for conflicts and redundancy.
2. Optimize context window usage by minimizing verbose rules.
3. Establish guidelines for memory management and context preservation.
