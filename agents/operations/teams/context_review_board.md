# Context Review Board

## Purpose
The Context Review Board is responsible for evaluating, validating, and approving changes to the core operational context, including `.clinerules`, `config/settings.md`, and global project constraints. The board ensures all rules scale properly, maintain technical soundness, and adhere to strict QA compliance.

## Members
- **Backend Architect**: Evaluates system impact, architectural integrity, and scalability of operational rules.
- **Senior Developer**: Assesses the technical soundness, developer experience, and practical workflow implications.
- **Evidence Collector (QA)**: Validates compliance, testability, and enforcement mechanisms of the defined rules.
- **Product Manager**: Ensures alignment with the product vision, Epic workflows, and overarching project goals.

## Responsibilities
- Conduct periodic reviews of global context files.
- Sign off on major Epic handoff workflows.
- Adjust routing rules and team operations to match current project phases.
- Validate that the memory maintenance protocols are enforceable and actively used.
