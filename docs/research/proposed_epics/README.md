# Proposed Epics Staging Area

## Purpose
This folder acts as a staging ground for drafted Product Requirements Documents (PRDs), Epic plans, and architectural proposals that have not yet been approved by the Ecosystem Review Board or added to the active development queue.

## Workflow Rule
According to the Standard Kickoff Protocol, documents must remain in this `research/proposed_epics` directory while they are being scoped and reviewed. 

They may only be moved into `docs/core/` once they receive official approval to commence development. This prevents unapproved drafts from being mistakenly interpreted as active marching orders by developer agents.
