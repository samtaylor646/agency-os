# Epic 3: Agent Connection & Orchestration (The Nexus Pipeline)

## Overview
Phase 3 focuses on bridging the plans generated by the Conversational Core Engine directly into agent execution. This relies on the Nexus Strategy to dynamically map tasks to agents, generate task queues, and provide real-time execution visibility. It also formally introduces the integration of custom agents and the document-driven pipeline.

## Objectives
1. **Dynamic Agent Selection**: Map task requirements directly to specific roles in the `/agents` directory.
2. **Custom Agent Integration (`agency-agents` format)**: Provide creation, parsing, and execution tools for custom specialized agents.
3. **Task Queue Generation**: Translate generated Markdown task lists into actionable items for `central_runner.py`.
4. **Execution Visibility**: Update the frontend Dashboard to display real-time logs and statuses of agents.
5. **Document Ingestion Pipeline**: Enable uploading existing documents (PDF, Markdown) to automatically seed the execution pipeline.

## Implementation Steps & Sprint Plan

### Step 1: Document Ingestion Pipeline
- Build an ingestion service to accept unstructured or structured documents (PDF, MD, TXT).
- Implement an LLM analysis pass to translate uploaded documents into a structured execution pipeline of tasks.
- Create a UI review step for users to approve the generated task list before execution.

### Step 2: Custom Agent Creator/Parser
- Create an interface or CLI tool to define new custom agents.
- Ensure custom agents use the `msitarzewski/agency-agents` format.
- Update `server/agent_parser.py` (if needed) to recognize and parse the new custom agent format seamlessly into the core pipeline.

### Step 3: Task Queue Translation & Dynamic Assignment
- Implement a parser that takes the generated Markdown task list and converts it into directed acyclic graph (DAG) nodes for `central_runner.py`.
- Build the logic to dynamically map task requirements to the appropriate agent from the `/agents` registry based on capabilities and system prompts.

### Step 4: Frontend Execution Visibility
- Update the React Dashboard (`client/src/AnalyticsDashboard.jsx` or new component) to transition from Planning to Execution mode.
- Stream real-time logs and status updates from `central_runner.py` to the UI via WebSockets or polling.

## Success Metrics
- Seamless execution of a generated task list from the Conversational Engine into the `central_runner.py` orchestrator.
- Successful parsing and execution of at least one user-uploaded document (e.g., a sample PRD).
- Successful registration and task assignment for a custom agent using the `agency-agents` format.
- Real-time visibility of task execution on the frontend dashboard.
